# Plan: Standardize `generate_resources` Return Type (E)

## Context
Datadog's `generate_resources` returns `Result(#(List(Resource), List(String)), CompilationError)` (resources + warnings), while the other 3 vendors return `Result(List(Resource), CompilationError)`. This forces `compiler.gleam` to wrap each non-Datadog vendor in a lambda that maps the result into a tuple with empty warnings: `|> result.map(fn(r) { #(r, []) })`. Standardizing the return type eliminates these wrappers and makes the `platform_for` function uniform.

## Changes

### 1. Update `generate_resources` in honeycomb, dynatrace, newrelic

Change return type from `Result(List(Resource), CompilationError)` to `Result(#(List(Resource), List(String)), CompilationError)`, wrapping the final `Ok` with empty warnings `[]`.

**honeycomb.gleam** (lines 29-36):
```gleam
// Before
Ok(list.flatten(resource_lists))
// After
Ok(#(list.flatten(resource_lists), []))
```

**dynatrace.gleam** (lines 29-33):
```gleam
// Before
irs |> list.try_map(ir_to_terraform_resource)
// After
irs |> list.try_map(ir_to_terraform_resource) |> result.map(fn(r) { #(r, []) })
```

**newrelic.gleam** (lines 32-36):
```gleam
// Before
irs |> list.try_map(ir_to_terraform_resource)
// After
irs |> list.try_map(ir_to_terraform_resource) |> result.map(fn(r) { #(r, []) })
```

### 2. Simplify `platform_for` in `compiler.gleam` (lines 97-141)

Remove the 3 wrapper lambdas. All 4 vendors now pass `vendor.generate_resources` directly, matching Datadog's pattern:

```gleam
vendor.Honeycomb ->
  VendorPlatform(
    vendor: vendor.Honeycomb,
    generate_resources: honeycomb.generate_resources,
    ...
  )
```

Same for Dynatrace and NewRelic.

### 3. No changes to `generate_terraform`

The public `generate_terraform` functions keep their current return types. Only the `@internal generate_resources` functions change. Tests call `generate_terraform`, not `generate_resources`, so no test changes needed.
